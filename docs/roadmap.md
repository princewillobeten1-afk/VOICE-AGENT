# Roadmap

## Phase 1: Foundation

- Establish monorepo structure.
- Define product architecture and domain boundaries.
- Build dashboard shell and design system foundation.
- Build API skeleton with typed contracts.
- Model AI employees, conversations, providers, tools, and logs.

## Phase 2: First Working Employee

- Create employee configuration flow.
- Add text conversation playground.
- Add provider adapter abstraction.
- Store conversations and structured events.
- Add basic observability views.

## Phase 3: Voice Runtime

- Add streaming voice session gateway.
- Support STT, LLM, and TTS provider adapters.
- Implement interruption and turn-taking behavior.
- Add phone provider integration.
- Track latency and runtime quality metrics.

## Phase 4: Developer Platform

- Publish REST API contract.
- Add API keys, webhooks, and examples.
- Add WebSocket voice/session APIs.
- Add SDK design and documentation.

## Phase 5: Enterprise Readiness

- Add roles, permissions, audit logs, SSO readiness, retention controls, and compliance posture.
- Add multi-region and scaling strategy.
- Add evaluation, testing, and reliability workflows.
