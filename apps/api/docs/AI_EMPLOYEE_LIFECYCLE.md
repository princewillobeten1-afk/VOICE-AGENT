# AI Employee Lifecycle

The AI Employee Builder turns the Task 011 architecture into a product workflow for hiring, training, testing, and publishing AI employees.

## Lifecycle Stages

1. Draft: identity, role, instructions, and configuration are being assembled.
2. Builder: the employee is moving through guided setup steps.
3. Testing: playground simulations and readiness checks are reviewed.
4. Published: the current version is approved for future runtime channels.
5. Archived: the employee is removed from active operations without deleting history.

## Core Operations

- Create employee
- Update profile
- Create version
- Publish version
- Duplicate employee
- Archive employee
- Export configuration
- Run playground simulation placeholder

## Runtime Boundary

Task 012 does not execute real AI, voice, RAG, tools, or workflows. It creates the management and configuration foundation that future runtime tasks will consume.
