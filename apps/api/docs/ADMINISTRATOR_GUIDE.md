# Administrator Guide

The Security workspace gives enterprise admins a single place to reason about access, governance, compliance, secrets, audit, and risk.

## Recommended Admin Workflow

1. Configure organization roles and groups.
2. Add SSO connection metadata.
3. Define MFA and trusted-device policy.
4. Add governance policies for AI employees, tools, workflows, knowledge, and memory.
5. Add ABAC policies for sensitive resource classes.
6. Register secret references and rotation rules.
7. Track compliance frameworks and evidence.
8. Review audit logs, sessions, risk events, and health recommendations.

## Caution

Readiness records are operational tools. They do not replace formal security review, legal review, penetration testing, vendor risk review, or compliance audits.
