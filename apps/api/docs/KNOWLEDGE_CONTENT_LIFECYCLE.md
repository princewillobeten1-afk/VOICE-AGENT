# Knowledge Content Lifecycle

1. Create knowledge base.
2. Register source.
3. Upload or create content.
4. Validate content and detect duplicates.
5. Organize by folders, categories, tags, and collections.
6. Draft content version.
7. Publish content version.
8. Sync or refresh source metadata.
9. Monitor freshness and quality.
10. Archive or delete stale content.

Content becomes RAG-ready only in a future retrieval task.
