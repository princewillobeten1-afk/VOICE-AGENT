# Tool Lifecycle

1. Tool is registered with name, category, schema, auth, permissions, timeout, retry policy, and metadata.
2. Initial tool version is created.
3. Tool is enabled, disabled, updated, or versioned by authorized users.
4. AI employees discover only available tools in their organization/workspace context.
5. Runtime validates inputs and permissions before execution.
6. Execution records and logs are persisted.
7. Analytics and health metrics summarize tool reliability.

Tools can be internal, integration-backed, custom, workflow actions, developer APIs, or future MCP-backed tools.
